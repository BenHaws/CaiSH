# CaiSH Challenge Submission Copy

## Project Title

CaiSH: AI-Assisted Treasury Command Center

## Short Description

CaiSH is an AI-assisted treasury operating system for global finance teams. It helps users monitor liquidity, stress-test cash positions, inspect cross-entity risk, and execute simulated treasury strategies with ledger-backed controls.

## 500-Character Description

CaiSH is an AI-assisted treasury command center for global finance teams. It unifies liquidity dashboards, payment-route monitoring, ISO 20022 ingestion, FX risk, working-capital stress tests, and Oil & Gas hedging workflows in one demo. Built with Codex, React, Express, SQLite, and quantitative simulations, it shows how treasury leaders can detect risk, model shocks, and execute ledger-backed strategy faster across subsidiaries, banks, commodities, and currencies with a polished glass UI today.

## Project Description

CaiSH addresses a real enterprise finance problem: treasury teams often manage fragmented bank accounts, regional subsidiaries, liquidity risks, and payment workflows across many systems. This project brings those workflows into one command center with an interactive React dashboard, an Express/SQLite backend, and quantitative simulation modules for liquidity forecasting, ALM duration matching, energy hedging, CVA-style exposure, working capital stress testing, and intercompany netting.

The app includes a global liquidity cockpit, scenario engine, industry-specific vertical modules, ISO 20022 ingestion, DEV/admin controls, and a strategy execution path that updates balances and records double-entry virtual ledger movements. I used Codex to help inspect the codebase, stabilize the TypeScript/Node toolchain, fix backend runtime issues, improve demo reliability, and prepare the project for submission.

## Tools Used

Codex, React, TypeScript, Vite, Express, Drizzle ORM, better-sqlite3, Zod, Recharts, MapLibre, lucide-react, esbuild.

## 3-Minute Demo Flow

1. Open CaiSH and show the executive liquidity dashboard.
2. Explain the global treasury problem: fragmented accounts, regional risk, and slow manual decisions.
3. Open the Nexus / entity view to show global subsidiaries and accounts.
4. Show the Scenario Engine or industry cockpit to demonstrate stress testing.
5. Open Cerebro / strategy execution story: AI-assisted recommendation leads to a simulated treasury movement.
6. Show that execution updates balances and is backed by ledger controls.
7. Close with why this matters: faster treasury decisions, clearer risk visibility, and portfolio-ready AI-assisted software.
